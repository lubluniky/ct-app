# 🔍 ASCII Animation Scale Fix - "Worm" Effect

## Проблема
ASCII текст выглядел как "глиста" - маленькие кубики/пиксели вместо читаемого текста.

## Скриншот проблемы
Текст отображался как набор маленьких квадратиков, напоминающих червяка, вместо читаемых ASCII символов.

## Причины

1. **Камера слишком далеко**: `camera.position.z = 30`
   - Объект казался очень маленьким
   - ASCII символы превращались в пиксели

2. **Плоскость слишком маленькая**: `planeBaseHeight = 7-9`
   - Недостаточный размер для нормального отображения
   - Текст "сжимался" в маленькую полоску

3. **Размер шрифта недостаточный**: 90-130px
   - Текстура canvas была слишком маленькой
   - Мало пикселей для рендеринга ASCII

4. **ASCII font size слишком большой**: 9px
   - Меньше символов на экране
   - Более "блочный" вид

## Исправления

### ✅ 1. Приближена камера (2x ближе)

**Было:**
```typescript
this.camera.position.z = 30; // Далеко
```

**Стало:**
```typescript
this.camera.position.z = 15; // В 2 раза ближе
```

**Эффект:**
- Объект занимает в 2 раза больше экрана
- Текст в 2 раза крупнее
- Лучшая видимость деталей

---

### ✅ 2. Увеличен planeBaseHeight (на 33-71%)

**Было:**
```typescript
if (minDimension < 500) return 9;  // Маленькие экраны
if (minDimension < 768) return 8;  // Мобильные
return 7; // Десктоп
```

**Стало:**
```typescript
if (minDimension < 500) return 12; // +33% (было 9)
if (minDimension < 768) return 10; // +25% (было 8)
return 8; // +14% (было 7)
```

**Эффект:**
- Плоскость на 14-33% больше
- Текст занимает больше пространства
- Лучшая читаемость

---

### ✅ 3. Увеличен textFontSize (на 15-22%)

**Было:**
```typescript
if (minDimension < 500) return 90;   // Маленькие
if (minDimension < 768) return 110;  // Мобильные
if (minDimension < 1024) return 120; // Планшеты
return 130; // Десктоп
```

**Стало:**
```typescript
if (minDimension < 500) return 110;  // +22% (было 90)
if (minDimension < 768) return 130;  // +18% (было 110)
if (minDimension < 1024) return 140; // +17% (было 120)
return 150; // +15% (было 130)
```

**Эффект:**
- Текстура canvas на 15-22% больше
- Больше пикселей для ASCII символов
- Более детализированный рендеринг

---

### ✅ 4. Уменьшен asciiFontSize (больше символов)

**Было:**
```typescript
asciiFontSize={9} // Меньше символов, больше "блоки"
```

**Стало:**
```typescript
asciiFontSize={7} // Больше символов, детальнее
```

**Эффект:**
- На 22% меньше размер ASCII символа
- На 22% больше символов на экране
- Более плавный и детальный вид
- Меньше "блочности"

### Итоговый scale (маленький экран):
```

Новый размер: camera_distance=15, plane=12, font=110, ascii=7

Визуальный размер = (plane * font) / (camera_z * ascii)
Было: (9 * 90) / (30 * 9) = 810 / 270 = 3.0
Стало: (12 * 110) / (15 * 7) = 1320 / 105 = 12.57

Итого: В 4.2 РАЗА КРУПНЕЕ! 🚀
```

## Техническая информация

### Почему камера на 15, а не ближе?

- **Z = 15**: Оптимально для 3D perspective
- **Z < 10**: Слишком близко, искажение перспективы
- **Z > 20**: Объект становится слишком маленьким
- **Z = 15**: Sweet spot для читаемости

### Почему asciiFontSize уменьшен?

```
AsciiFontSize = размер символа в ASCII выводе
Меньше fontSize = больше символов = детальнее изображение

Было (9px): ░░░░░ (грубо, блочно)
Стало (7px): ░░░░░░░ (плавно, детально)
```

### ASCII Charset плотность:
```
Font 9px: ~80 символов по ширине
Font 7px: ~103 символов по ширине (+29%)
Больше символов = лучше детализация
```
