# Инструкция по настройке Nginx для Cloudflare Flexible SSL

## Контекст
Мы настроили домен `api.borkiss.trade` через Cloudflare.
В Cloudflare включен режим **Flexible SSL**.
Это значит, что Cloudflare принимает HTTPS запросы от пользователей, но передает их на наш сервер по **HTTP (порт 80)**.

## Задача
Настроить Nginx на сервере (91.107.193.27) как Reverse Proxy, который будет:
1. Слушать порт **80**.
2. Проксировать все запросы на локальный порт Rust приложения (**8080**).
3. Корректно обрабатывать WebSocket соединения.

## Конфигурация Nginx

1. Установите Nginx (если еще нет):
   ```bash
   sudo apt update
   sudo apt install nginx
   ```

2. Создайте конфиг для сайта:
   ```bash
   sudo nano /etc/nginx/sites-available/screener-api
   ```

3. Вставьте следующую конфигурацию:

   ```nginx
   server {
       listen 80;
       server_name api.borkiss.trade;

       location / {
           # Проксирование на Rust приложение
           proxy_pass http://127.0.0.1:8080;
           
           # Настройки для WebSocket
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection "upgrade";
           
           # Передача реальных IP адресов и заголовков
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```

4. Активируйте сайт:
   ```bash
   sudo ln -s /etc/nginx/sites-available/screener-api /etc/nginx/sites-enabled/
   sudo rm /etc/nginx/sites-enabled/default  # Удалить дефолтный конфиг, если мешает
   ```

5. Проверьте конфиг и перезапустите Nginx:
   ```bash
   sudo nginx -t
   sudo systemctl restart nginx
   ```

## Проверка
После настройки:
- `http://api.borkiss.trade/api/v1/health` должен открываться и возвращать JSON.
- `https://api.borkiss.trade/api/v1/health` (через Cloudflare) тоже должен работать.

## Важно
Не нужно настраивать SSL сертификаты (Let's Encrypt) на самом сервере, так как шифрование берет на себя Cloudflare. Сервер должен просто принимать HTTP на 80 порту.
